What is Universal Design?

Universal Design is the design/composition that anybody regardless of age, disability or any related factors can use it.

Norman's Design Principles


Affordance

Affordance is the idea that something matches the intended use. For example, a door knob is intended to be pulled and turned.


Constraint

The purpose of constraint is to prevent the user from misusing and causing error in an action. An example of this is when opening the door of a washing machine, the washer stops to prevent a mess being made.


Visibility

Visibility is tied with constraint as physical constraints being visible is most successful to prevent a wrong attempt even before trying it.


Mapping

Mapping is the idea that something within proximity of the operation is closely tied to it whether its due to the distance or position. A great example of this is the knobs on a stove which should be intuitive which belongs to which stove.


ARIA


Accessible Rich Internet Applications

ARIA is a set of attributes you can add to HTML elements that define ways to make web content and applications accessible to users with disabilities who use assistive technologies (AT)". Part of Universal Design includes ARIA which is something made for screen reader users and those who rely on keyboard. -Kat Shaw (lullabot)

Support for ARIA

Not all technologies support ARIA but those that do include browsers, AT (screen readers, magnifiers, etc), applications and JavaSciprt toolkits.


The Main Components of ARIA

Roles are a component of ARIA which are used to define a UI element

States and Properties are applied to elements and used to help ARIA roles.
